# PRD: Public Data Explorer

## 개요

**제품명:** Public Data Explorer  
**버전:** 0.0.0  
**작성일:** 2026-06-09

공공데이터포털(data.go.kr)에서 제공하는 공공기관 API·파일 데이터 목록 전체를 브라우저에서 탐색·검색·필터링할 수 있는 단일 페이지 애플리케이션이다.

---

## 배경 및 목적

공공데이터포털은 약 95,000건 이상의 데이터 목록을 보유하고 있으나, 포털 자체의 검색 UI가 느리고 필터 조합이 제한적이다. 본 프로젝트는 원천 CSV 파일(`공공데이터활용지원센터_공공데이터포털 목록개방현황_20260331.csv`)을 전처리하여 로컬 JSON으로 제공하고, 빠른 클라이언트 사이드 검색·필터링 경험을 제공하는 것을 목표로 한다.

---

## 사용자

- 공공데이터를 활용하는 개발자·연구자·정책 담당자
- 특정 기관이나 분류체계의 데이터를 빠르게 조회하고 싶은 일반 사용자

---

## 기능 요구사항

### 1. 데이터 로드

| 항목 | 내용 |
|------|------|
| 데이터 소스 | `public/data_full.json` (정적 JSON, 빌드 시 포함) |
| 레코드 수 | 약 95,623건 |
| 로드 방식 | 앱 초기화 시 `fetch('/data_full.json')` 단일 요청 |
| 로딩 상태 | 로딩 중 스피너 메시지 표시 (`데이터를 불러오는 중입니다...`) |

**데이터 필드:**

| 필드명 | 설명 |
|--------|------|
| 목록명 | 데이터셋 이름 |
| 파일데이터명 | 실제 파일 이름 |
| 제공기관 | 데이터 제공 기관명 |
| 분류체계 | 공공행정/보건의료/문화관광 등 대분류-소분류 |
| 목록유형 | FILE / API / STD 등 |
| 확장자(데이터포맷) | csv, json, xml 등 (복수 시 콤마 구분) |
| 키워드 | 검색 키워드 태그 |
| 설명 | 데이터 설명 |
| 수정일 | 최종 수정 일자 |
| 조회수 | 포털 조회수 |
| 다운로드수 | 다운로드/활용신청 건수 |
| 비용유무 | 무료 / 유료 |
| 목록 URL | 공공데이터포털 상세 페이지 링크 |

---

### 2. 상단 검색바 (TopBar)

- 전체 화면 상단 고정 헤더
- 로고(Database 아이콘 + "Public Data Explorer" 텍스트) 좌측 배치
- 텍스트 검색 입력창 (`데이터 목록 검색...` placeholder)
- **검색 대상 필드:** 목록명, 파일데이터명, 키워드, 설명
- **1글자 입력 시 검색 비활성화** (전체 데이터 표시), 2글자 이상부터 필터링
- `useDeferredValue`를 통해 입력 성능 최적화

---

### 3. 사이드바 필터 (Sidebar)

좌측 고정 패널에 4개의 접이식(accordion) 필터 그룹 제공:

| 필터 | 데이터 필드 | 설명 |
|------|------------|------|
| 목록유형 | `목록유형` | FILE, API, STD 등 |
| 제공기관 | `제공기관` | 전체 기관 목록 알파벳 정렬 |
| 분류체계 | `분류체계` | 공공행정, 보건의료 등 |
| 데이터 포맷 | `확장자(데이터포맷)` | csv, json, xml 등 (콤마 분리 파싱) |

- 각 그룹은 체크박스 다중 선택 지원
- 그룹 헤더 클릭 시 접기/펼치기 (기본: 펼침)
- 상단 "초기화" 버튼으로 모든 필터와 검색어 일괄 리셋
- 필터 옵션은 전체 데이터 기반 동적 생성 (`useMemo`)

---

### 4. 대시보드 (Dashboard)

데이터 목록 상단에 차트 카드 2개 표시 (데이터 로드 완료 후 렌더링):

#### 4-1. 분류체계별 데이터 비중 (Pie Chart)
- 분류체계별 레코드 수 집계
- 상위 10개 분류 표시, 나머지는 "기타"로 합산
- 색상 팔레트: 공공데이터 느낌의 블루 계열

#### 4-2. 상위 조회수 데이터 Top 10 (Bar Chart)
- 조회수 기준 상위 10개 데이터셋
- 목록명 15자 절사 후 `...` 처리
- 세로 막대 차트, 범례 숨김

---

### 5. 데이터 그리드 (DataGrid)

필터·검색 결과를 테이블 형태로 표시:

| 컬럼 | 필드 |
|------|------|
| 목록명 | `목록명` |
| 제공기관 | `제공기관` |
| 분류체계 | `분류체계` |
| 확장자 | `확장자(데이터포맷)` 첫 번째 값, 배지(badge) 스타일 |
| 조회수 | `조회수` 숫자 포맷 |
| 수정일 | `수정일` |

- **페이지네이션:** 페이지당 20건, 처음/이전/다음/맨뒤 버튼
- 필터·검색 변경 시 1페이지로 자동 리셋
- 검색 결과 없을 경우 "검색 결과가 없습니다." 표시
- 총 건수 헤더에 표시 (`총 N건`)
- 행 클릭 시 상세 패널 오픈

---

### 6. 상세 패널 (DetailPanel)

행 클릭 시 오른쪽에서 슬라이드인하는 사이드 패널:

- 오버레이 배경 클릭 또는 X 버튼으로 닫기
- 표시 항목: 목록명, 제공기관, 분류체계, 목록유형, 확장자(배지), 설명, 수정일, 조회수, 다운로드수, 비용유무
- "공공데이터포털에서 보기" 외부 링크 버튼 (`목록 URL`, `target="_blank"`)

---

## 데이터 전처리 스크립트

원천 CSV → `public/data_full.json` 변환을 담당하는 Node.js 스크립트:

- **파일:** `process_data.cjs`
- **입력:** `공공데이터활용지원센터_공공데이터포털 목록개방현황_20260331.csv`
- **출력:** `public/data_full.json` (약 95,623 레코드)
- 필요한 컬럼만 추출하여 파일 크기 최소화
- 멀티바이트(EUC-KR) CSV 처리를 위해 `iconv-lite` 사용

## 성능 고려사항

| 기법 | 적용 위치 | 목적 |
|------|-----------|------|
| `useDeferredValue` | 검색어 | 타이핑 중 UI 블로킹 방지 |
| `useMemo` | 필터 옵션 생성, 필터링 결과, 차트 데이터 | 불필요한 재계산 방지 |
| `React.memo` | DataGrid, Dashboard | 불필요한 재렌더링 방지 |
| 1글자 검색 비활성화 | 검색 필터 | 단일 자모 입력 시 전체 결과 불필요하게 줄어드는 문제 방지 |
| 클라이언트 사이드 렌더링 | 전체 | 서버 없이 정적 배포 가능 |

---

## 비기능 요구사항

- 외부 API 의존성 없음 (완전 정적 배포 가능)
- 빌드 산출물은 정적 파일(HTML/JS/CSS/JSON)로만 구성
- 모든 데이터는 공공데이터포털 공개 목록 기반 (개인정보 없음)
